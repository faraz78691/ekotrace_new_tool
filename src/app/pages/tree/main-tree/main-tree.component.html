Organisation Tree

There is a three-layer organizational structure for emissions reporting, with the following hierarchy:

Top Layer: Main Group (Parent Company)
  1. - No emissions data input at this level
  2. - Only aggregated data from sub-groups is reported
Mid Layer: Sub-Groups (Sub-Entities)
  1. - Financed emissions data is input and reported at this level
  2. - Sub-groups are subsidiaries or business units of the main group
  3. - Emissions data from facilities is aggregated to this level
Lower Layer: Facilities
  1. - GHG Emissions data is input for each facility. Some examples of facilities - office, manufacturing plant, sales store, data centre.
  2. - Facilities are associated with sub-groups and report emissions data to their respective sub-group
  3. -Data is then aggregated from facilities to sub-groups and finally to the main group
This structure allows for:
  1. Granular emissions tracking at the facility level
  2. Aggregated reporting at the sub-group level, including financed emissions
  3. Consolidated reporting at the main group level, providing an overall emissions profile
How to create your organisation tree
  1. Load template - Choose one of the three available templates
    • - Single Corporate
    • - Corporate with multiple sub-entities
    • - VC fund
  2. Edit Node - Add, delete, or edit nodes in the organizational structure. Nodes represent entities in the hierarchy, such as: Main Group, Sub-Groups, Facilities.
    • - Top layer (Main Group) – There can only be one node which represents the Parent company name.
    • - Middle layer (Sub-groups) – The nodes represent subsidiaries or business units of the main group. To add a ‘sub-group’ node, right click on Main group node and enter sub-group details like subsidiary name
    • - Bottom-layer (Facilities) – To add facilities, right click on sub-group node and add facility details including facility name and facility type e.g. office, mfg. plant, sales store
  3. Ensure the tree structure follows the predefined hierarchy
    • - Main Group (top-level)
    • - Sub-Groups (mid-level)
    • - Facilities (lower-level)
